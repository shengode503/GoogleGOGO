# Section 3 - Google Coding Rules

For every problem:

- [ ] Read the problem carefully.
- [ ] Spend at least 20-30 minutes thinking before asking for hints.
- [ ] Explain the brute-force solution first.
- [ ] Explain why your chosen data structure fits.
- [ ] Discuss time complexity before coding.
- [ ] Write clean, readable code.
- [ ] Use meaningful variable names.
- [ ] Dry run with at least one normal test case.
- [ ] Dry run with at least one edge case.
- [ ] Analyze Time Complexity.
- [ ] Analyze Space Complexity.
- [ ] Think of at least one possible Google follow-up.
- [ ] Consider alternative solutions after finishing.

# Section 5 - Google Interview Notes

Remember during every interview:

- Communicate before coding.
- Clarify assumptions.
- Think aloud.
- Discuss trade-offs.
- Handle edge cases.
- Write production-quality code.
- Test your solution.
- Stay calm when receiving follow-up questions.

**Goal:** Don't just solve the problem—demonstrate how you think.
